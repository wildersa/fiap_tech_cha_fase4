Entenda o Pipeline LSTM do Tech Challenge

Uma visão curta, visual e direta do que o projeto faz: dados históricos entram, a LSTM aprende padrões temporais e a API retorna o próximo fechamento previsto.

1 · Problema

O que estamos tentando prever?

O modelo tenta prever o fechamento do próximo pregão de uma ação, usando uma janela de valores anteriores.

últimos N pregões → LSTM → próximo fechamento
2 · Entrada

O que vira X?

X é a janela temporal. Exemplo: se window_size = 60, o modelo recebe os últimos 60 passos da série.

No modo single, a API usa fechamentos e calcula retornos quando necessário.

3 · Alvo

O que vira y?

y é o valor que o modelo aprende a prever. Pode ser preço bruto, retorno simples ou log-retorno.

No modo log_returns, o modelo prevê a variação logarítmica e depois reconverte para preço.

Baseline

Por que comparar?

Em preço de ação, um chute forte é:

preço de amanhã = fechamento de hoje

Esse é o baseline_last_close. A LSTM precisa vencer essa regra simples para mostrar ganho real.

MAE

Erro em reais

Mostra quanto o modelo erra, em média, na escala do preço.

MAE = média(|real - previsto|)
RMSE

Pune erros grandes

Fica mais alto quando existem previsões muito ruins em alguns dias.

RMSE = raiz(média(erro²))
MAPE

Erro percentual

É a métrica mais fácil de explicar na apresentação.

MAPE 1,2% ≈ erro médio de 1,2%
Sem vazamento

Regra crítica do scaler

Certo: dividir os dados no tempo, fitar o scaler no treino e só transformar validação/teste.

Errado: fitar o scaler no dataset inteiro antes do split. Isso deixa o treino “ver” o futuro.

Atenção

O ponto mais importante hoje

A API simplificada aceita apenas feature_mode="single". Modelos com technical_features, ohlcv ou ohlcv_returns são bons para experimento, mas não devem ser promovidos se o endpoint continuar recebendo só fechamentos.

Resumo para apresentar

Explicação em 30 segundos

O projeto usa uma rede LSTM para prever o fechamento do próximo pregão da PETR4. A entrada é uma janela temporal com pregões anteriores, e o alvo é o próximo fechamento ou sua variação. Os dados são divididos cronologicamente em treino, validação e teste, e a normalização é ajustada apenas no treino para evitar vazamento. O modelo é treinado com MSE, AdamW, early stopping e gradient clipping. A avaliação é feita em preço real usando MAE, RMSE e MAPE, sempre comparando contra o baseline de último fechamento.
